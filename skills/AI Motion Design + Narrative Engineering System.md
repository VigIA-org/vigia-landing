- gradients
- colors
- animations
- glassmorphism

- visual narrative
- cinematic experience
- guided interaction system
- motion-driven storytelling interface

- alive
- immersive
- intentional
- fluid
- emotionally directed

The interface must guide attention using:
- motion hierarchy
- pacing
- rhythm

# Visual Principles

## 1. Motion Hierarchy

Not all elements should animate equally.

Define:
- what enters first
- what has visual priority
- what attracts attention
- what remains secondary

Animations must guide the eye intentionally.

---

## 2. Smoothness First

Prioritize:
- smoothness
- rhythm
- elegant easing
- subtle transitions

Over:
- excessive movement
- flashy effects
- aggressive animations

---

## 3. Invisible Complexity

The best motion design is often barely noticeable consciously.

The goal is:
"this feels premium"

NOT:
"look at this animation"

---

# Required Motion Systems

## Fade + Slide Reveal

Elements should:
- start with low opacity
- begin slightly translated
- smoothly reveal on viewport entry

Recommended:
- y: 20px–40px
- opacity: 0 → 1
- duration: 0.6s–1s

---

## Stagger Animations

Related elements should NEVER appear simultaneously.

Use:
- stagger timing
- progressive delays

Especially for:
- cards
- statistics
- feature lists
- logos
- text groups

---

## Scroll-Linked Motion

Animations should react to scroll progression.

Use:
- translate
- scale
- rotate
- opacity shifts

to create:
- physicality
- depth
- cinematic movement

---

## Soft Parallax

Use subtle parallax layers:
- gradients
- backgrounds
- glow effects
- floating images

Movement must feel:
- elegant
- slow
- immersive

Avoid exaggerated parallax.

---
